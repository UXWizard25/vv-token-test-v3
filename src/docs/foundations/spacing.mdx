## Semantic Spacing

Kontextbasierte Abstände für spezifische Anwendungsfälle:

### Grid Spacing

| Token | Wert | Verwendung |
|-------|------|------------|
| `--grid-space-resp-base` | 12-24px | Basis Grid Gap (responsive) |
| `--grid-margin` | 16-48px | Außenrand des Grids |
| `--grid-gutter` | 16-24px | Abstand zwischen Spalten |

### Content Spacing

| Token | Wert | Verwendung |
|-------|------|------------|
| `--content-gap-sm` | 8px | Kleine Abstände |
| `--content-gap-md` | 16px | Standard Abstände |
| `--content-gap-lg` | 24px | Große Abstände |
| `--content-gap-xl` | 32px | Extra große Abstände |

### Component Spacing

| Token | Verwendung |
|-------|------------|
| `--button-inline-space` | Horizontaler Button-Innenabstand |
| `--button-stack-space` | Vertikaler Button-Innenabstand |
| `--card-padding` | Card Innenabstand |
| `--input-padding` | Input Field Padding |

## Density Modes

Spacing passt sich an den Density-Modus an:

| Mode | Faktor | Verwendung |
|------|--------|------------|
| `default` | 1.0x | Standard UI |
| `dense` | 0.75x | Datentabellen, Listen |
| `spacious` | 1.25x | Hero Sections, Marketing |

```html
<!-- Density via data-Attribut -->
<html data-density="default">  <!-- Standard -->
<html data-density="dense">    <!-- Kompakt -->
<html data-density="spacious"> <!-- Großzügig -->
```

> **Tipp:** Wechsle den Density-Modus im Toolbar um die Unterschiede zu sehen.

---

## Usage

```css
/* Spacing Tokens verwenden */
.my-component {
  padding: var(--space-2-x);           /* 16px */
  margin-bottom: var(--space-3-x);     /* 24px */
  gap: var(--content-gap-md);          /* 16px */
}

/* Responsive Grid */
.grid {
  display: grid;
  gap: var(--grid-space-resp-base);    /* 12-24px je nach Breakpoint */
  padding: var(--grid-margin);
}
```

---

## Best Practices

1. **Konsistenz:** Verwende immer Spacing-Tokens statt feste Pixelwerte
2. **Semantik:** Bevorzuge semantische Tokens (`--content-gap-md`) vor primitiven (`--space-2-x`)
3. **Density:** Teste UI in allen drei Density-Modi
4. **Responsive:** Nutze responsive Tokens für automatische Anpassung
